线性分类

在监督学习的范畴中，分类 (Classification) 是最核心和最常见的一类任务。这里将从分类问题的基本概念出发，详细介绍最基础的线性分类模型。

信用卡例子

在信用卡审批的例子中，假设我们将每个申请人的信息（例如，薪水和工龄）作为两个特征，绘制在一个二维平面上。历史上被批准的申请人，我们用蓝色“+”号表示；被拒绝的，用红色“-”号表示。
在这个场景下，每一个数据点(xi,yi)(\boldsymbol{x}_i, y_i)(xi​,yi​)就构成了一个训练样本，其中：
  • xi\boldsymbol{x}_ixi​是第iii个申请人的特征向量（例如，xi=[薪水i,工龄i]⊤\boldsymbol{x}_i = [\text{薪水}_i, \text{工龄}_i]^{\top}xi​=[薪水i​,工龄i​]⊤）。
  • yiy_iyi​是对应的标签，在这个例子中yi∈{+1,−1}y_i \in \{+1, -1\}yi​∈{+1,−1}，其中+1+1+1代表“批准”，−1-1−1代表“拒绝”。
给定一组训练数据样本(x1,y1)…,(xn,yn)(\boldsymbol{x}_1,y_1)\dots,(\boldsymbol{x}_n,y_n)(x1​,y1​)…,(xn​,yn​)，我们的目标是学习一个模型（或函数）fθf_{\boldsymbol{\theta}}fθ​，它能够将特征空间中的任意一点映射到一个离散的标签。具体到这个二元分类 (binary classification) 问题，模型的映射关系为：
fθ:Rd→{−1,+1}f_{\boldsymbol{\theta}}: \mathbb{R}^d \rightarrow \{-1, +1\}fθ​:Rd→{−1,+1}
这里的ddd是特征的数量（在我们的例子中d=2d=2d=2）。这个学习到的模型fθf_{\boldsymbol{\theta}}fθ​是对那个未知的、理想的决策规律ggg的一个近似。
那么，我们应该选择什么样形式的模型fθf_{\boldsymbol{\theta}}fθ​呢？最直观、最简单的想法就是在特征空间中画一条直线，将两个类别的点分开。
这条红色的直线就是一个决策边界 (Decision Boundary)。所有落在直线上方的点，我们预测为“+1”；所有落在下方的点，我们预测为“-1”。这种试图用一个线性方程来划分数据类别的模型，就是线性分类模型 (Linear Classification Model) 。

感知器 (The Perceptron)

感知机是历史上第一个被提出的机器学习算法之一，由 Frank Rosenblatt 于1957年提出。它不仅是神经网络的雏形，也是理解更复杂分类算法的基石。

模型定义

模型的输出fθ(x)f_{\boldsymbol{\theta}}(\boldsymbol{x})fθ​(x)是一个连续的实数值，它代表了点x\boldsymbol{x}x距离决策边界的远近和方向。但它并不是我们最终需要的离散分类标签+1 +1 +1或+1 +1 +1。如何转换？
为了完成从连续得分到离散类别的转换，我们需要在模型中引入一个非线性组件，这个组件在神经网络的语境下被称为激活函数 (Activation Function) 。
感知机所使用的激活函数是符号函数 (Sign Function)，它是一种形式非常简洁的阶跃函数 (Step Function)。它的作用是：当输入大于某个阈值时，输出一个值；当输入小于阈值时，输出另一个值。
我们设置阈值为000，即通过判断fθ(xi)f_{\boldsymbol{\theta}}(\boldsymbol{x}_i)fθ​(xi​)的正负号来进行决策：
  • 如果fθ(xi)＞0f_{\boldsymbol{\theta}}(\boldsymbol{x}_i) ＞ 0fθ​(xi​)＞0，说明该点在决策边界的正侧，我们预测其类别为+1 +1 +1。
  • 如果fθ(xi)<0f_{\boldsymbol{\theta}}(\boldsymbol{x}_i) < 0fθ​(xi​)<0，说明该点在决策边界的负侧，我们预测其类别为−1 -1 −1。
  • 如果fθ(xi)=0f_{\boldsymbol{\theta}}(\boldsymbol{x}_i) = 0fθ​(xi​)=0，说明该点恰好在决策边界上。
（注：数学上，阶跃函数的值一般是000或111，这里使用−1-1−1和+1+1+1）
结合线性和激活两个部分，一个完整的感知机模型可以被定义为：

至此，我们已经完整构建了感知机模型。它接收一个输入x\boldsymbol{x}x，并能够输出一个预测的分类标签。

学习算法

在定义了模型之后，我们需要设计一个算法，根据训练数据来找到最优的分类器，即找到那组最优的权重参数θ\boldsymbol{\theta}θ。
为了理解算法背后的直觉，我们需要先明确权重向量θ\boldsymbol{\theta}θ的几何意义：
在几何上，方程θ⊤x=0\boldsymbol{\theta}^{\top}\boldsymbol{x} = 0θ⊤x=0定义了一个超平面（即决策边界）。而权重向量θ\boldsymbol{\theta}θ正是这个超平面的法向量 (Normal Vector)。以二维分类模型为例，方程θ⊤x=0\boldsymbol{\theta}^{\top}\boldsymbol{x} = 0θ⊤x=0是一条直线（θ1x1+θ2x2+b=0\theta_{1}x_{1}+\theta_{2}x_{2}+b=0θ1​x1​+θ2​x2​+b=0），向量θ\boldsymbol{\theta}θ的方向必然垂直于这条直线。因此，我们改变θ\boldsymbol{\theta}θ，本质上是对这条直线的进行旋转。
几何规律：一个完美的分类器是怎样的？假设下图呈现的是一个完美的分类器，我们可以发现一个几何规律，对于正类样本，与θ\boldsymbol{\theta}θ的夹角是锐角；对于负类样本，则与θ\boldsymbol{\theta}θ的夹角是钝角。
基于这个几何规律，我们希望通过不断调整θ\boldsymbol{\theta}θ的方向，使其满足以下几何关系：
  1. 对于所有正类样本(yi=+1y_i = +1yi​=+1)，我们要让θ\boldsymbol{\theta}θ与xi\boldsymbol{x}_ixi​的夹角变为锐角（即方向大致相同），从而保证θ⊤xi>0\boldsymbol{\theta}^{\top}\boldsymbol{x}_i > 0θ⊤xi​>0。
  2. 对于所有负类样本(yi=−1y_i = -1yi​=−1)，我们要让θ\boldsymbol{\theta}θ与xi\boldsymbol{x}_ixi​的夹角变为钝角（即方向大致相反），从而保证θ⊤xi<0\boldsymbol{\theta}^{\top}\boldsymbol{x}_i < 0θ⊤xi​<0。
基于这样的直觉，我们只需要把样本一个个地拿出来，看一下当前样本xi\boldsymbol{x}_ixi​与当前θ\boldsymbol{\theta}θ的夹角：
  • 针对被错误分类的样本：如果xi\boldsymbol{x}_ixi​是正类样本，夹角却是钝角，或者xi\boldsymbol{x}_ixi​是负类样本，夹角却是锐角，就说明模型在这个样本上犯错了，我们需要旋转θ\boldsymbol{\theta}θ来修正这个错误。
  • 如何旋转：①当xi\boldsymbol{x}_ixi​是正类样本，与θ\boldsymbol{\theta}θ的夹角却是钝角，我们就让θnew=θ+xi\boldsymbol{\theta}_{\text{new}} = \boldsymbol{\theta} + \boldsymbol{x}_{i}θnew​=θ+xi​，这会把θ\boldsymbol{\theta}θ的方向推往xi\boldsymbol{x}_ixi​的方向；②当xi\boldsymbol{x}_ixi​是负类样本，与θ\boldsymbol{\theta}θ的夹角却是锐角，我们就让θnew=θ−xi\boldsymbol{\theta}_{\text{new}} = \boldsymbol{\theta} - \boldsymbol{x}_{i}θnew​=θ−xi​，这会把θ\boldsymbol{\theta}θ的方向推往xi\boldsymbol{x}_ixi​的反方向。
可以说，感知机是一种错误驱动 (error-driven) 的在线学习算法：
  • 模型只有在犯错的时候才会进行调整和学习。
  • 如果一个样本已经被正确分类，模型则不进行任何改变。
上述流程似乎很复杂，需要我们不断地判断是“正”还是“负”，但感知机的算法流程非常简洁。

  2. 送代：循环遍历整个训练数据集。在每一次迭代中，当一个样本的预测结果与真实标签不符时，即sign⁡(θ⊤xi)≠yi\operatorname{sign}(\boldsymbol{\theta}^\top\boldsymbol{x}_i)\neq y_isign(θ⊤xi​)=yi​。这个条件等价于yi(θ⊤xi)≤0y_i(\boldsymbol{\theta}^\top\boldsymbol{x}_i)\leq0yi​(θ⊤xi​)≤0。

    θ:=θ+yixi\boldsymbol{\theta}:=\boldsymbol{\theta}+y_i\boldsymbol{x}_iθ:=θ+yi​xi​

算法特性

理解了算法流程后，我们还需要深入探讨它的一些关键特性和局限性。
  • 收敛性 (Convergence)
    如果训练数据集是线性可分 (linearly separable)的（即，确实存在一个超平面能将数据完美分开），那么感知机算法保证能在有限次的迭代后收敛，并找到这样一个超平面。
  • 最优性 (Optimality)
    感知机找到的解并非最优。对于一个线性可分的数据集，可能存在无数个满足条件的超平面。感知机只会满足于找到任意一个可以成功划分数据的解，然后就停止了。它不会去寻找那个“最好”的、最“鲁棒”的决策边界（例如，距离两个类别的最近点都有最大间隔的那个）。这个问题将由后续的支持向量机 (SVM) 来解决。
  • 线性不可分问题 (The Non-Linearly Separable Case)
    这是感知机算法的一个致命弱点。如果数据集本身就不是线性可分的，那么算法将永远无法找到一个能将所有点都正确分类的解，因此算法将永不收敛，会一直在数据点之间来回振荡。在实际应用中，需要使用其他算法（如带有“口袋”机制的感知机）或更复杂的非线性模型来处理这类问题。

  • 泛化能力 (Generalization)
    假设感知机成功地在训练集上找到了一个完美的分割线，这是否意味着它在未来的、未知的测试数据上也会表现良好？答案是不一定。模型在训练数据上的表现好，不等于它的泛化能力强。这引出了机器学习中一个核心的议题：过拟合 (Overfitting) 与泛化。

逻辑回归

条件概率与分类

我们的任务是建立一个分类器，例如信用卡审批申请，结果只有两种：“批准 (Approve)” 或 “拒绝 (Reject)”。在机器学习中，我们将这些文字标签转换为数字。这里，“批准”对应y=+1y = +1y=+1，“拒绝”对应y=−1y = -1y=−1。
现在有一个新的申请数据x\boldsymbol{x}x（x\boldsymbol{x}x可以包含申请人的各种信息，如收入、信用记录等），但它没有标签。我们需要判断应该将它分为 “批准” 还是 “拒绝”。
假设我们通过某种方式“神奇地”知道了以下两个概率：
  • Pr⁡[y=+1∣x]=0.8\Pr[y = +1|\boldsymbol{x}] = 0.8Pr[y=+1∣x]=0.8：给定数据x\boldsymbol{x}x的条件下，这个申请被“批准”的概率是80%。
  • Pr⁡[y=−1∣x]=0.2\Pr[y = -1|\boldsymbol{x}] = 0.2Pr[y=−1∣x]=0.2：给定数据x\boldsymbol{x}x的条件下，这个申请被“拒绝”的概率是20%。
那么答案显而易见，既然“批准”的概率（80%）远大于“拒绝”的概率（20%），最合理的选择就是将x\boldsymbol{x}x归类为“批准” (y=+1y = +1y=+1)。

从二分类到多分类

逻辑回归是解决二元分类问题的经典模型。例如，判断一封邮件是否是垃圾邮件（是/否）。它通过Sigmoid函数将模型的线性输出映射到(0,1)(0,1)(0,1)区间，这个值可以被解释为属于其中一个类别的概率。
然而，在现实世界中，我们经常遇到需要将事物分到两个以上类别的情况，即多分类问题。例如：
  • 手写数字识别：将一张图片识别为 0, 1, 2, ..., 9 中的某一个数字。
  • 新闻文章分类：将一篇文章归类为体育、财经、科技、娱乐等类别。
  • 图像内容识别：判断一张图片里的动物是猫、狗，
当分类任务的类别K>2 K>2 K>2时，逻辑回归就无法直接应用了。Softmax 回归（Softmax Regression），有时也被称为多项逻辑回归（Multinomial Logistic Regression），正是为了解决这类问题而设计的。

Softmax函数

解决多元分类问题的核心思想非常直观：既然一个模型只能做二元判断，那我们就为每个类别都建立一个专属的“评判标准”。如果总共有K K K个类别，我们就会有K K K套独立的权重向量，我们将其表示为θ1,θ2,…,θK\boldsymbol{\theta}_{1}, \boldsymbol{\theta}_{2}, \ldots, \boldsymbol{\theta}_{K}θ1​,θ2​,…,θK​。每一个权重向量θk\boldsymbol{\theta}_{k}θk​都对应着第k k k个类别。
对于一个给定的输入样本xi\boldsymbol{x}_ixi​，模型会执行以下操作：
  • 为每个类别计算一个“分数”（Score）：模型使用每个类别的权重向量θk\boldsymbol{\theta}_{k}θk​与输入样本xi\boldsymbol{x}_ixi​进行线性运算（点积），为该类别生成一个分数。这个分数可以被理解为模型认为样本xi\boldsymbol{x}_ixi​属于类别k k k的“证据”或“倾向性”有多强。
    score(xi,k)=θk⊤xi\text{score}(\boldsymbol{x}_i, k) = \boldsymbol{\theta}_k^\top \boldsymbol{x}_iscore(xi​,k)=θk⊤​xi​
  • 将分数转换为概率：得到所有KK K个类别的分数后，我们需要一种方法将这些任意大小的分数（可以是正数、负数或零）转换成一个有效的概率分布。这个概率分布必须满足两个条件：
    • 每个类别的概率值都在 0 和 1 之间。
    • 所有类别的概率值加起来总和为 1。
完成这个转换任务的函数，就是Softmax 函数。
Softmax 函数是整个模型的核心。假设我们有一个包含KK K个分数的向量s=[s1,s2,…,sK]⊤\mathbf{s}=\left[s_{1}, s_{2}, \ldots, s_{K}\right]^{\top}s=[s1​,s2​,…,sK​]⊤，其中sks_{k}sk​是第k k k个类别的分数。Softmax 函数会对这个向量进行如下操作，计算出第k k k个类别的概率：
Softmax(s)k=esk∑j=1Kesj\text{Softmax}(s)_k = \frac{e^{s_k}}{\sum_{j=1}^K e^{s_j}}Softmax(s)k​=∑j=1K​esj​esk​​
这个函数做了两件事：
  1. 指数化：它使用指数函数eske^{s_k}esk​将所有的分数都映射到正数域，确保概率值为正。
  2. 归一化：它将指数化后的分数除以所有类别指数化分数之和（∑j=1Kesj\sum_{j=1}^Ke^{s_j}∑j=1K​esj​）。这保证了所有计算出的概率值加起来正好等于 1。
这完美地符合概率分布的定义。Softmax 是 Sigmoid 在多分类场景下的自然推广。此外，Softmax 这个名字的含义是“平滑化的最大值（soft arg max）”。它输出的概率分布会倾向于给分数最高的那个类别一个远大于其他类别的概率值。

逻辑回归总结

模型：如何用数学函数模拟概率？
核心概念: 逻辑回归（LR）最关键的思想就是使用逻辑函数 (logistic function / sigmoid) 来近似后验概率Pr⁡[y∣x]\Pr[y|\boldsymbol{x}]Pr[y∣x]。对于二分类问题，这个模型可以被简洁地写为：
Pr⁡[y∣x]←Pr⁡θ[y∣x]=11+exp⁡(−y⋅θ⊤x)\Pr[y|\boldsymbol{x}] \leftarrow \Pr_{\boldsymbol{\theta}}[y|\boldsymbol{x}] = \frac{1}{1 + \exp(-y \cdot \boldsymbol{\theta}^\top \boldsymbol{x})}Pr[y∣x]←θPr​[y∣x]=1+exp(−y⋅θ⊤x)1​
LR 直接使用带标签的训练数据{(xi,yi)} \{(\boldsymbol{x}_i, y_i)\} {(xi​,yi​)}来学习出这个概率模型的参数θ \boldsymbol{\theta}θ。
当类别数大于2时，我们将 Sigmoid 函数推广为 Softmax 函数。Softmax 为每个类别都学习一个权重向量，并能保证所有类别的预测概率之和为1，完美地满足了多分类的概率建模需求。
对于更复杂的模型如深度神经网络 (deep neural networks) 和语言模型 (language models)，其根本目标之一也是学习Pr⁡[y∣x]\Pr[y|\boldsymbol{x}]Pr[y∣x]，但它们实现这一目标的方式更为复杂和间接。
目标函数 (Objective Function)：我们要解决的学习问题，就是最小化在训练集上的平均损失，这个损失函数（以二元逻辑回归为例）记为L(θ)\mathcal{L}(\boldsymbol{\theta})L(θ)：
L(θ):=1n∑i=1nlog⁡(1+exp⁡(−yi⋅θ⊤xi))\mathcal{L}(\boldsymbol{\theta}):=\frac1n\sum_{i=1}^n\log(1+\exp(-y_i\cdot\boldsymbol{\theta}^\top \boldsymbol{x}_i))L(θ):=n1​i=1∑n​log(1+exp(−yi​⋅θ⊤xi​))
我们的目标是找到能使这个损失函数最小化的参数θ^\hat{\boldsymbol{\theta}}θ^：
θ^=argmin⁡θ∈RdL(θ)\hat{\boldsymbol{\theta}}=\underset{\boldsymbol{\theta}\in\mathbb{R}^d}{\operatorname*{argmin}}\mathcal{L}(\boldsymbol{\theta})θ^=θ∈Rdargmin​L(θ)
  • 坏消息 ❎：这个问题没有解析解 (No closed-form solution)。意味着我们不能像解线性回归那样，用一个简单的数学公式直接算出θ^\hat{\boldsymbol{\theta}}θ^。但是这个目标函数L(θ)\mathcal{L}(\boldsymbol{\theta})L(θ)是一个关于参数θ \boldsymbol{\theta}θ的凸函数 (convex function)。
  • 好消息 ✅：一个凸函数就像一个碗，它只有一个全局最低点，没有任何局部的坑洼（局部最小值）。这意味着，我们只要顺着“碗”壁往下滑，就一定能滑到碗底的唯一最低点。因此，我们可以使用梯度下降 (gradient descent)等优化算法，来逐步逼近并最终找到这个全局最优解θ^\hat{\boldsymbol{\theta}}θ^，而不必担心被困在某个非最优的局部解中。
因此，逻辑回归的学习问题是一个没有解析解、但性质优良（凸函数）的优化问题，因此可以用梯度下降等算法来有效求解。
